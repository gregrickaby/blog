Sass. What is it?

  • NOUN
    1) Impudent or disrespectful back talk. "My mother refuses to take any sass from me."
  • VERB
    2) To answer back in an impudent manner. "Don't sass mom."
  • STYLESHEET LANGUAGE
    3) Sass is a stylesheet language, that, when compiled, writes traditional CSS.

Brief History

  • Sass was initially designed in 2007 by Hampton Catlin and developed by Nathan Weizenbaum.
  • The orginal markup, .sass was modeled after another language named, "Haml" and uses indentation to seperate CSS blocks.
  • The newer markup, .scss closley models traditional CSS blocks and indentation, making the transition from CSS to Sass easier.
  • Sass runs on Ruby and has influenced other stylesheet languages, such as: LESS, Stylus, and Tritium.

For the remainder of this presentation, I will concentrate on the .scss markup

Misconceptions

Misconsceptions

To install Sass I needed to install Gems, and to install Gems I needed to install Ruby, and to install Ruby I need to Grunt and sudo the oskeychain helper...

No!! You can download one of many apps (which I will highlight in a minute), point, click, and be compiling in minutes. No command line needed.

I don't have Admin access to my web server. So, why even bother learning Sass?

Sass lives and breathes on your local machine. It doesn't do anything (nor does it even belong on a web server). Browsers can't read or even interperate Sass. This is why Sass is a PRE-processor. It's processed before being uploaded to your web server.

Basic File Structure

Sass and it's stylesheets reside inside your theme's directory.

/_s

  
/images

  
/js

  
/sass

     
/partials

        
_content.scss
      <-- partial stylesheet
        
_header.scss
       <-- partial
        
_navigation.scss
   <-- partial
        
_mixins.scss
       <-- partial (mixins)
        
_sidebar.scss
      <-- partial
        
_variables.scss
    <-- partial (variables)
        
_widgets.scss
      <-- partial
    
config.rb
    <-- config file for compiler
    
style.scss
   <-- non-partial primary SASS stylesheet
  
index.php

  
style.css
   <-- the compiled stylesheet

What is a Compiler

  • A compiler is simply a program, that when ran, turns one thing into another.

  • In this case, it compiles style.scss into style.css.
  • Without a compiler? Sass doesn't "do" anything.

Compiler Workflow

The compiler can "watch" for changes to your Sass directory.

  • Anytime you save your changes, the compiler will compile - and update your style.css ...automatically!
  • If you're developing locally and using LiveReload, then your browser would refresh showing the udpated style
  • At that point you would upload/push style.css to the web server.

Compiler Config File

This is a basic configuration file (config.rb) which provides instructions to the compiler. Without it, the compiler wouldn't know where to look - or where to output the CSS.

sass_dir      = "/"           <-- Sass stylesheet directory
http_path     = "../"         <-- Path to your website
css_dir       = "../"         <-- CSS directory (the output of compiler)
output_style  = expanded      <-- Output style (can be "compact" or "expanded")
line_comments = false         <-- Helpful during development
  • I could speak at ad nauseam about the different ways to setup config.rb.
  • Reference: http://compass-style.org/help/tutorials/configuration-reference/

The Lingo

Partials

"Partials" are simply partial stylesheets which contain little snippets of CSS. This is a great way to modularize your CSS and help keep things easier to maintain.

  • A partial is simply a stylesheet starting with a leading underscore. You might name it something like _typography.scss.
  • Break up theme and content parts into a content heirarchy: _header.scss, _content.scss, _sidebar.scss, _footer.scss, etc...
  • The leading underscore lets the compiler know, "Hey, I'm only one piece of the puzzle!"

Partials

Partial stylesheets are included into the primary Sass stylesheet via the @import directive. Here is an example style.scss

/*
Theme Name: _s
Theme URI: http://underscores.me/
Author: Automattic
Author URI: http://automattic.com/
*/
@import "variables";  
<-- Notice we don't have to include the leading _

@import "mixins";     
<-- or the .scss file extension

@import "header";
@import "typography";
@import "content";
@import "footer";

Variables

Think of variables as a way to store information that you want to reuse throughout your stylesheets. You can store things like colors, font stacks, or any CSS value you'd want to re-use throughout your theme. Just like PHP, Sass uses the $ symbol.

body
 {
  background-color: #f5f5f5;
  color: #111;
  font-family: 'Open Sans', sans-serif;
  font-size: 16px;
}

Above is some very basic CSS. Instead of writing #f5f5f5; over and over and over...

Variables

Let's use a variable! First, create _variables.scss

_variables.scss



$color-background
: #f5f5f5;

$color-font
: #111;

$font-sans
: "Open Sans", sans-serif;

Now create a new file named, _defaults.scss and plug in your new variables:

body
 {
  background-color: 
$color-background;

  color: 
$color-font;

  font-family: 
$font-sans;

}

Variables

After being compiled, the end result would be:

body
 {
  background-color: #f5f5f5;
  color: #111;
  font-family: 'Open Sans', sans-serif;
}

Nesting

When you write HTML, you probably use a fairly clear nested, visual hierarchy with indentation. With CSS? Nesting can get confusing in a hurry. Sass lets you nest your CSS selectors, in the same visual hierarchy of HTML.

nav ul
 {
  list-style: none;
}


nav ul li
 {
  display: inline-block;
}


nav ul li a
 {
  color: #111;
  text-decoration: none;
}
nav
 {
   
ul
 {
      list-style: none;
      
li
 {
         display: inline-block;
        
a
 {
           color: #111;
           text-decoration: none;
        }
      }
   }
}

Nesting

When compiled, it looks like:

nav ul
 {
  list-style: none;
}


nav ul li
 {
  display: inline-block;
}


nav ul li a
 {
  color: #111;
  text-decoration: none;
}

Nesting

Use an ampersand & as a "shorthand" way to include a parent.

.stupid-long-selector-name
 {
   background: $color-background;
    
&:hover
 {
      background: $color-hover;
   }
}

.stupid-long-selector-name
 {
   background: #f5f5f5;
    
.stupid-long-selector-name:hover
 {
        background: #d3d3d3;
    }
}

Import

  • The CSS @import feature allows you to import several stylesheets into one. The caveat however, is each @import creates a new HTTP request.
  • But Sass is compiled locally remember? Sass builds on top of the current CSS @import function and combines everything into a single CSS file.

Mixins

Some things in CSS are a bit tedious to write, especially vendor prefixes. A mixin lets you make groups of CSS declarations, or "functions" that you could reuse throughout your theme. You can even pass in arguments! Here's an example for border-radius.

_mixin.scss



@mixin border-radius($radius)
 {
  -webkit-border-radius: $radius;
     -moz-border-radius: $radius;
      -ms-border-radius: $radius;
          border-radius: $radius;
}

_content.scss



.box
 {
   @include border-radius(10px);
}

Mixins

When compiled, style.css shows:

.box
 {
  -webkit-border-radius: 10px;
     -moz-border-radius: 10px;
      -ms-border-radius: 10px;
          border-radius: 10px;
}
  • If the thought of writing a bunch of mixins seems a bit overwhelming, don't worry! Mixins are included with Sass frameworks such as Compass or Bourbon. More on those later...

Extend/Inheritance

Using @extend lets you share a set of CSS properties from one selector to another. Think of @extend like variables, only they're chunks of code.

.message
 {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}


.success
 {
  @extend .message;
  border-color: green;
}


.error
 {
  @extend .message;
  border-color: red;
}

.message
 {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}


.success
 {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
  border-color: green;
}


.error
 {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
  border-color: red;
}

Operators

Doing math in your CSS is very helpful. Sass has a handful of standard math operators like +, -, *, /, and %. Here is a simple fluid grid, based on 960px:

.container
 { width: 100%; }


article[role="main"]
 {
  float: left;
  width: 600px / 960px * 100%;
}


aside[role="complimentary"]
 {
  float: right;
  width: 300px / 960px * 100%;
}

.container
 { width: 100%; }


article[role="main"]
 {
  float: left;
  width: 62.5%;
}


aside[role="complimentary"]
 {
  float: right;
  width: 31.25%;
}

Frameworks

Because Sass is all about writing less code, and just like theme frameworks for WordPress (_s, Genesis, Hybrid, etc...) are about giving you a head start; there are a few Sass frameworks (namely Compass and Bourbon) that provide a ton of mixins and snippets.

Some clarifications about Compass and Compass.app. Compass.app is simply the name of a compiler - that comes pre-bundled with the Compass Framework.

Frameworks

Installing frameworks can be done via the command line. (Unless it comes pre-bundled, then it's already there!)

Compass gem install compass

Bourbon gem install bourbon

Neat (requires Bourbon) gem install neat

Bitters (requires Bourbon) gem install bitters

Frameworks

Including a framework is as easy as using @import. Now we have all kinds of mixins available for our project!

style.scss


@import "compass";
@import "variables"
@import "mixins"    etc...

_content.scss



.foo
 {
	color: tint( $color, $percentage );
	@include transition( all 0.8s );
}
